# 创意工坊 — 界面设计文档

> 日期：2026-05-30
> 版本：v1.0
> 基于：novel-assistant 现有 CLI 系统

---

## 1. 任务理解

### 1.1 目标

将现有的 CLI 小说创作记忆助手扩展为 **FastAPI Web 界面**，核心价值是：

> **输入构思 → 召回相关记忆 → AI 生成建议 → 格式化为候选记忆 → 用户确认入库**

### 1.2 约束

- 系统仍然是 **创作记忆库**，不是自动写作平台
- 不保存章节正文，不生成完整章节
- 复用现有 `memory.py` / `retrieve.py` / `suggest.py` / `llm.py` 模块
- 前端首版使用 **FastAPI + Jinja2 + 原生 JavaScript**，不引入前端框架

### 1.3 预期输出

- 一个 FastAPI 应用，启动后浏览器打开即可使用
- 包含灵感箱、大纲、人物、场景、剧情、伏笔、世界观、全部记忆等页面
- 灵感箱支持完整的"构思 → 检索 → 建议 → 确认"闭环

## 2. 整体布局

采用 **左侧导航 + 中央工作区 + 右侧面板** 的三栏工作台布局。

```
┌─────────────────────────────────────────────────────────────┐
│  顶部工具栏：项目名 | 记忆总数 | 向量库状态 | LLM 状态      │
├──────────┬──────────────────────────────┬───────────────────┤
│          │                              │                   │
│  左侧    │      中央工作区               │   右侧面板        │
│  导航    │                              │                   │
│          │  （根据选中页面切换内容）      │  - 相关记忆       │
│ · 灵感箱 │                              │  - AI 建议        │
│ · 大纲   │                              │  - 候选记忆       │
│ · 人物   │                              │  - 确认入库       │
│ · 场景   │                              │                   │
│ · 剧情   │                              │                   │
│ · 伏笔   │                              │                   │
│ · 世界观 │                              │                   │
│ · 全部   │                              │                   │
│          │                              │                   │
├──────────┴──────────────────────────────┴───────────────────┤
│  底部状态栏（可选）                                          │

```

### 布局说明

| 区域 | 宽度 | 用途 |
|------|------|------|
| 左侧导航 | 固定 200px | 页面切换，始终可见 |

| 右侧面板 | 固定 320px | 灵感箱模式下显示检索结果、AI建议、候选记忆；其他模式下显示选中条目的详情 |

```
│                                  │                  │

│  │ 多行文本输入框              │  │  · 人物: 顾北辰  │
│  │ （输入你的构思...）         │  │    害怕海水...    │
│  │                            │  │  · 剧情: 第三章   │
│  │                            │  │    出现红怀表...  │
│  └────────────────────────────┘  │                  │
│                                  │  [AI 建议]       │
│  类型提示：○自动判断 ○大纲 ○人物   │  1. 可以加入PTSD  │
│            ○场景 ○剧情 ○伏笔      │  2. 安排雨夜回忆   │
│            ○世界观               │  3. 关联父亲失踪   │
│                                  │                  │
│  [检索记忆]  [生成建议]  [清空]    │  [候选记忆]       │
│                                  │  ┌──────────────┐│
│  ── AI 建议区域 ──               │  │ type: scene   ││
│                                  │  │ title: ...    ││
│  建议1: 可以加入PTSD反应          │  │ content: ...  ││
│  建议2: 安排雨夜回忆场景          │  │ tags: [...]   ││
│  建议3: 关联父亲失踪事件          │  │ [编辑] [删除] ││
│                                  │  └──────────────┘│
│  ── 候选记忆预览 ──              │                  │
│                                  │  [确认入库]       │
│  ┌────────────────────────────┐  │                  │
│  │ type: scene                │  │                  │
│  │ title: 雨夜码头触发回忆     │  │                  │
│  │ content: 顾北辰在雨夜...    │  │                  │
│  │ tags: [码头, 红怀表]       │  │                  │
│  │ [编辑] [删除]              │  │                  │
│  └────────────────────────────┘  │                  │
│                                  │                  │
└──────────────────────────────────┴──────────────────┘
```

#### 交互流程

```
用户输入构思
    │
    ▼
点击 [检索记忆] ──→ 调用 MemoryRetriever.search_memories()
    │                    │
    ▼                    ▼
右侧显示相关记忆     按类型分组展示
    │
    ▼
点击 [生成建议] ──→ AI 返回 suggestions + candidate_memories
    │                    │
    ▼                    ▼
中央显示建议列表    右侧显示候选记忆卡片
    │
    ▼
用户编辑候选记忆（类型/标题/内容/标签）
    │
    ▼
点击 [确认入库] ──→ MemoryManager.add_memory() + 向量库更新
    │
    ▼
提示成功，清空候选区
```

#### 关键原则

- **AI 不能自动写入记忆**，必须经过用户确认
- 确认前必须允许用户编辑候选记忆的所有字段
- 相关记忆必须可见，确保 AI 建议不脱离已有设定
- 建议可以一键转为候选记忆（点击建议旁的"+"按钮）

## 4. 记忆类型扩展

现有系统支持 `character` / `plot` / `callback` 三种类型。界面设计扩展为以下类型：

| 类型 | type 值 | 用途 | 关键字段 |
|------|---------|------|----------|
| 大纲 | `outline` | 宏观结构、卷纲、章节计划 | `title`, `content`, `tags` |
| 人物 | `character` | 人物设定、动机、变化 | `name`, `content`, `tags` |
| 场景 | `scene` | 地点、氛围、冲突、出场人物 | `title`, `content`, `tags` |
| 剧情 | `plot` | 剧情点、事件、转折 | `title`, `content`, `tags` |
| 伏笔 | `callback` | 伏笔、线索、悬念、回调 | `title`, `content`, `tags` |
| 世界观 | `worldbuilding` | 规则、组织、历史、术语 | `title`, `content`, `tags` |
| 笔记 | `note` | 暂未归类的灵感或素材 | `title`, `content`, `tags` |

### 兼容策略

- 旧记忆没有 `title` / `tags` / `status` 时正常读取，显示 `content` 前 30 字作为标题
- `character` 继续使用 `name` 字段
- `MemoryType` 枚举扩展，但 `type` 字段仍为字符串，保持向后兼容

### 扩展后的数据结构

```json
{
  "id": "uuid",
  "type": "scene",
  "title": "雨夜码头重逢",
  "name": null,
  "content": "顾北辰在雨夜码头再次看见红怀表，触发童年事故记忆。",
  "source": "用户原始输入片段",
  "tags": ["码头", "红怀表", "顾北辰"],
  "status": "confirmed",
  "created_at": "2026-05-30T12:00:00",
  "updated_at": "2026-05-30T12:00:00"
}
```

## 5. 后端 API 设计

### 5.1 页面路由

```text
GET  /                       → 灵感箱（首页）
GET  /outline                → 大纲
GET  /character              → 人物
GET  /scene                  → 场景
GET  /plot                   → 剧情
GET  /callback               → 伏笔/回调
GET  /worldbuilding          → 世界观
GET  /memories               → 全部记忆
```

### 5.2 API 路由

```text
# 记忆 CRUD
GET    /api/memories                    → 获取记忆列表（支持 ?type= &search= 参数）
GET    /api/memories/{id}               → 获取单条记忆
POST   /api/memories                    → 新增记忆
PUT    /api/memories/{id}               → 更新记忆
DELETE /api/memories/{id}               → 删除记忆

# 灵感箱专用
POST   /api/inspirations/search         → 检索相关记忆
       Body: { "query": "用户构思文本", "type": "auto"|"outline"|... }
       Response: { "memories": [...] }

POST   /api/inspirations/advise         → 生成建议和候选记忆
       Body: { "query": "用户构思文本", "related_memories": [...] }
       Response: {
         "suggestions": ["建议1", "建议2", ...],
         "candidate_memories": [{ "type": "...", "title": "...", "content": "...", "tags": [...] }]
       }

POST   /api/inspirations/confirm        → 确认入库候选记忆
       Body: { "memories": [{ "type": "...", "title": "...", "content": "...", "tags": [...] }] }
       Response: { "saved": 3, "ids": ["uuid1", "uuid2", "uuid3"] }

# 工具
POST   /api/vector/sync                 → 同步向量库
GET    /api/stats                       → 记忆统计
GET    /api/export                      → 导出全部记忆为 JSON
```

### 5.3 API 响应格式

统一使用：

```json
{
  "code": 0,
  "message": "success",
  "data": { ... }
}
```

错误时：

```json
{
  "code": 1,
  "message": "错误描述",
  "data": null
}
```

## 6. AI 输出格式

灵感箱中 AI 的输出拆为两个部分：

```json
{
  "suggestions": [
    "可以把红怀表作为顾北辰童年事故的触发物。",
    "码头场景适合保留潮湿、噪声和闪回三类感官线索。"
  ],
  "candidate_memories": [
    {
      "type": "scene",
      "title": "雨夜码头触发回忆",
      "content": "顾北辰在雨夜码头看到红怀表，引发童年事故相关记忆。",
      "tags": ["顾北辰", "码头", "红怀表"]
    }
  ]
}
```

**容错**：如果 AI 返回的 JSON 格式不合法，界面显示原始文本，并提示用户手动创建记忆。

## 7. 技术实现方案

### 7.1 技术栈

| 层 | 技术 | 说明 |
|----|------|------|
| Web 框架 | FastAPI | 异步、自动文档、Jinja2 支持 |
| 模板 | Jinja2 | 服务端渲染 HTML |
| 前端 | 原生 JavaScript | fetch API 调用后端接口 |
| 样式 | 内联 CSS 或单文件 CSS | 保持简单，不引入框架 |
| 后端逻辑 | 复用现有模块 | memory.py / retrieve.py / suggest.py / llm.py |

## 8. 错误处理

| 场景 | 处理方式 |
|------|----------|
| LLM 未配置 | 顶部显示警告横幅，"生成建议"按钮禁用 |
| 向量库未初始化 | 允许加载记忆列表，检索时提示"请先同步向量库" |
| Embedding 模型加载慢 | 检索按钮显示加载动画，避免界面无反馈 |
| AI JSON 解析失败 | 保留建议文本，不自动创建候选记忆，提示用户手动操作 |
| 保存失败 | 候选记忆保留在页面，不清空，显示错误信息 |
| 网络超时 | 显示超时提示，提供重试按钮 |

## 9. 实现边界

### ✅ 本阶段做

- FastAPI Web 入口 + Jinja2 模板渲染
- 三栏工作台布局（左侧导航 + 中央工作区 + 右侧面板）
